作用域和闭包

作用域

==> 用来管理标识符可以在什么情况下被访问和修改的一套规则。

  • JS引擎不会上来就开始执行每一行代码。
  • 首先 编译器 会逐行解析代码，并且对每一行代码进行编译
  • 先找到 var 和 function 这两个标识符
  • 然后：
  1. 问作用域，当前作用域注册过 变量 a 么？
  2. 作用域 告诉 编译器 没有
  3. 那么 就声明一个变量 a，并且把 var 声明的 变量提升到代码最前面
  4. 然后发现没有其它 var 或者 funciton 了
  5. 编译完成每行代码，JS引擎开始逐行执行

作用域 分为 全局作用域和局部作用域（通常只函数作用域）

作用域链

当作用域互相嵌套的时候，那么会形成作用域链。

作用域链：规定了局部作用域可以访问外部作用域的标识符，但是外部作用域不能访问局部作用域里面的标识符





var a = 1;
 function fn1() {
  fn2()
  function fn2 () {
   console.log(a + 1);
   }
}

JS存放数据主要分为3个部分：

  1. 静态区域：主要存放全局的变量，只要浏览器不关闭或者不手动指向null，那么这个变量就会一直存在。

  2. 栈：凡是可以被自动释放的变量都存放在这里

  3. 堆：复杂的数据类型一般都存放在这里，不会被自动释放

      • 浏览器的JS引擎都存在 垃圾回收 机制，由垃圾回收器定时释放
